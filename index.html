What is Go & Why Learn It?

History, philosophy, and use cases of Go vs JavaScript.

▾
ℹ️
Go (Golang) was created at Google in 2009 by Robert Griesemer, Rob Pike, and Ken Thompson. It was designed to solve real-world problems at Google scale — fast compilation, easy concurrency, and simplicity.

JavaScript Is Great For:

  • Frontend/browser apps
  • Rapid prototyping
  • Dynamic/loosely typed code
  • Event-driven I/O (Node.js)

Go Is Great For:

  • Microservices & APIs
  • CLI tools & system software
  • High-performance backends
  • Cloud-native / Kubernetes
💡
Companies using Go: Google, Uber, Docker, Kubernetes, Cloudflare, Dropbox, Twitch. Go powers most of the cloud-native ecosystem.

📌

• Every Go file must start with package name
• The entry point is always func main() in package main
• Unused imports = compile error (unlike JS)
• fmt = "format" — Go's print/format package

Variables, Types & Zero Values

Go is statically typed. No undefined — everything has a default (zero) value.

▾
⚡ JavaScript
// Dynamic — no types
let name = "Alice"
let age = 30
let score = 9.5
let active = true
let nothing   // undefined

// age = "thirty" ← allowed!
🐹 Go
// Statically typed
var name string = "Alice"
var age int = 30
var score float64 = 9.5
var active bool = true
var nothing int  // = 0 (zero value!)

// Short declaration (preferred)
city := "Dhaka"  // type inferred
💡
Go Zero Values (no undefined/null surprises):
int → 0  |  float64 → 0.0  |  string → ""  |  bool → false  |  pointers/slices/maps → nil
🐹 All Basic Types
// Integers
var a int     // platform-sized (64-bit on 64-bit OS)
var b int8    // -128 to 127
var c int64   // -9 quadrillion to 9 quadrillion
var d uint    // unsigned (no negatives)

// Floats
var f1 float32
var f2 float64  // prefer this

// Text
var s string    // UTF-8 encoded
var r rune      // alias for int32, represents Unicode char
var by byte     // alias for uint8

// Constants (compile-time, cannot change)
const Pi = 3.14159
const AppName = "MyApp"

Control Flow — if, for, switch

Go only has for loops — no while, no do-while.

▾
⚡ JavaScript
// if/else
if (age >= 18) {
  console.log("adult")
} else {
  console.log("minor")
}

// for loop
for (let i = 0; i < 5; i++) {
  console.log(i)
}

// while
while (x > 0) { x-- }
🐹 Go
// if/else (no parentheses!)
if age >= 18 {
  fmt.Println("adult")
} else {
  fmt.Println("minor")
}

// Classic for
for i := 0; i < 5; i++ {
  fmt.Println(i)
}

// "while" in Go
for x > 0 { x-- }

// Infinite loop
for { /* runs forever */ }
🐹 Switch & if with init statement
// if with init statement (unique to Go!)
if err := doSomething(); err != nil {
  fmt.Println("Error:", err)
}
// err is scoped to this if block only

// Switch (no break needed, no fallthrough by default)
switch day {
case "Mon", "Tue":
  fmt.Println("Weekday")
case "Sat", "Sun":
  fmt.Println("Weekend")
default:
  fmt.Println("Unknown")
}

Functions — Multiple Returns & Named Results

Go functions can return multiple values — this is how errors are handled.

▾
⚡ JavaScript
// Single return
function add(a, b) {
  return a + b
}

// "Multiple returns" via object
function divide(a, b) {
  if (b === 0)
    return { result: null, error: "divide by zero" }
  return { result: a / b, error: null }
}

// Arrow functions
const square = (n) => n * n
🐹 Go
// Typed params & return
func add(a, b int) int {
  return a + b
}

// Multiple returns (idiomatic!)
func divide(a, b float64) (float64, error) {
  if b == 0 {
    return 0, fmt.Errorf("cannot divide by zero")
  }
  return a / b, nil
}

// Usage
result, err := divide(10, 2)
if err != nil {
  log.Fatal(err)
}

// Variadic function
func sum(nums ...int) int {
  total := 0
  for _, n := range nums {
    total += n
  }
  return total
}
🚨
In Go, error handling is explicit. You ALWAYS check err != nil after every operation that can fail. This is the #1 difference from JS try/catch patterns. Ignoring errors is a code smell.

Arrays, Slices & Maps

Slices are like JS arrays. Maps are like JS objects/Maps.

▾
⚡ JavaScript
// Array (dynamic)
const nums = [1, 2, 3]
nums.push(4)
nums.length  // 4
nums[1:3]   // not valid in JS

// Object / Map
const user = { name: "Alice", age: 30 }
user.city = "Dhaka"
delete user.age
🐹 Go
// Array (fixed size, rarely used)
var arr [3]int = [3]int{1, 2, 3}

// Slice (dynamic, use this!)
nums := []int{1, 2, 3}
nums = append(nums, 4)
len(nums)         // 4
nums[1:3]         // [2 3] — slicing!

// Map
user := map[string]interface{}{
  "name": "Alice",
  "age":  30,
}
user["city"] = "Dhaka"
delete(user, "age")

// Check if key exists
val, ok := user["name"]
if ok { fmt.Println(val) }
🐹 Range — iterating (like forEach)
// Range over slice
fruits := []string{"apple", "banana", "mango"}
for i, fruit := range fruits {
  fmt.Printf("%d: %s\n", i, fruit)
}

// Range over map
scores := map[string]int{"Alice": 95, "Bob": 87}
for name, score := range scores {
  fmt.Printf("%s scored %d\n", name, score)
}

// _ to ignore index or value
for _, fruit := range fruits { fmt.Println(fruit) }

Structs — Go's "Classes"

Go has no classes. Structs + Methods replace them. This is Go's OOP.

▾
⚡ JavaScript Class
class Person {
  constructor(name, age) {
    this.name = name
    this.age = age
  }
  greet() {
    return `Hi, I'm ${this.name}`
  }
}
const p = new Person("Alice", 30)
p.greet()
🐹 Go Struct + Method
type Person struct {
  Name string
  Age  int
}

// Method on struct
func (p Person) Greet() string {
  return "Hi, I'm " + p.Name
}

// Constructor pattern
func NewPerson(name string, age int) *Person {
  return &Person{Name: name, Age: age}
}

p := NewPerson("Alice", 30)
p.Greet()

Interfaces — Implicit & Powerful

Unlike TypeScript, Go interfaces are satisfied implicitly — no implements keyword.

▾
⚡ TypeScript Interface
interface Animal {
  speak(): string
}

// Must explicitly implement
class Dog implements Animal {
  speak() { return "Woof" }
}

class Cat implements Animal {
  speak() { return "Meow" }
}
🐹 Go Interface (implicit!)
type Animal interface {
  Speak() string
}

// Dog satisfies Animal automatically
type Dog struct{}
func (d Dog) Speak() string { return "Woof" }

type Cat struct{}
func (c Cat) Speak() string { return "Meow" }

// Polymorphism!
func makeSound(a Animal) {
  fmt.Println(a.Speak())
}
💡
Key insight: If a type has the same methods as an interface, it automatically satisfies it. This enables amazing decoupling and testability. The most important Go interface is error — it just requires one method: Error() string

Pointers — The Key to Mutation

JS passes objects by reference automatically. In Go, you choose when to use a pointer.

▾
🐹 Pointer Basics
func main() {
  x := 42
  p := &x          // p is a *int (pointer to x)
  fmt.Println(*p)  // 42 — dereference
  *p = 100         // modify x through pointer
  fmt.Println(x)   // 100
}

// Without pointer — copy, original unchanged
func noChange(n int) { n = 999 }

// With pointer — modifies original
func withChange(n *int) { *n = 999 }

// Pointer receiver — modify struct in method
func (p *Person) Birthday() {
  p.Age++  // actually modifies the struct
}
📌
Rule of thumb:
• Use pointer receiver *Type when the method needs to modify the struct
• Use value receiver Type when you just need to read
• Large structs should use pointer receivers for performance (avoids copying)
⚡

Goroutines & Channels — Go's Superpower

Go's concurrency model is revolutionary. Goroutines are like ultra-light threads.

▾
⚡ JavaScript async/await
// JS is single-threaded
async function fetchData() {
  const res = await fetch(url)
  return res.json()
}

// "Parallel" via Promise.all
await Promise.all([fetch1(), fetch2()])
🐹 Goroutines (true parallel!)
// Launch goroutine with 'go'
go doWork()   // runs concurrently!

// Channel — safe communication
ch := make(chan string)

go func() {
  ch <- "Hello from goroutine"
}()

msg := <-ch  // receive (blocks until ready)
fmt.Println(msg)
🐹 WaitGroup — wait for goroutines
import "sync"

func main() {
  var wg sync.WaitGroup

  for i := 0; i < 5; i++ {
    wg.Add(1)
    go func(id int) {
      defer wg.Done()
      fmt.Printf("Worker %d done\n", id)
    }(i)
  }

  wg.Wait()  // blocks until all goroutines finish
  fmt.Println("All workers done!")
}
💡
Go's concurrency mantra: "Do not communicate by sharing memory; share memory by communicating." — use channels to pass data between goroutines instead of mutexes (when possible).

Error Handling — The Go Way

No try/catch. Errors are values returned from functions.

▾
🐹 Custom Errors & Wrapping
import "errors" // Sentinel errors var ErrNotFound = errors.New("not found") // Custom error type type ValidationError struct { Field string Message string } func (e *ValidationError) Error() string { return e.Field + ": " + e.Message } // Error wrapping (Go 1.13+) func getUser(id int) (*User, error) { user, err := dbQuery(id) if err != nil { return nil, fmt.Errorf("getUser(%d): %w", id, err) } return user, nil } // Unwrapping errors if errors.Is(err, ErrNotFound) { ... } var ve *ValidationError if errors.As(err, &ve) { ... }

Generics (Go 1.18+)

Write functions and types that work with any type safely.

▾
🐹 Generic Functions
// Before generics (using interface{})
func contains(slice []interface{}, item interface{}) bool {
  for _, v := range slice {
    if v == item { return true }
  }
  return false
}

// With Generics (Go 1.18+) — type safe!
func Contains[T comparable](slice []T, item T) bool {
  for _, v := range slice {
    if v == item { return true }
  }
  return false
}

// Works with any type!
Contains([]int{1, 2, 3}, 2)          // true
Contains([]string{"a", "b"}, "a")  // true

// Generic Stack
type Stack[T any] struct {
  items []T
}
func (s *Stack[T]) Push(item T) { s.items = append(s.items, item) }
func (s *Stack[T]) Pop() T {
  n := len(s.items) - 1
  item := s.items[n]
  s.items = s.items[:n]
  return item
}

Testing — Built Into Go

No jest/mocha needed. go test is built-in, fast, and powerful.

▾
🐹 Writing Tests
// math.go
func Add(a, b int) int { return a + b }

// math_test.go (same package, _test.go suffix)
import "testing"

func TestAdd(t *testing.T) {
  result := Add(2, 3)
  if result != 5 {
    t.Errorf("expected 5, got %d", result)
  }
}

// Table-driven tests (idiomatic Go)
func TestAddTable(t *testing.T) {
  tests := []struct{ a, b, want int }{
    {1, 2, 3},
    {0, 0, 0},
    {-1, 1, 0},
  }
  for _, tt := range tests {
    got := Add(tt.a, tt.b)
    if got != tt.want {
      t.Errorf("Add(%d,%d) = %d; want %d", tt.a, tt.b, got, tt.want)
    }
  }
}

# Run tests
# go test ./...
# go test -v -cover ./...
# go test -bench=. ./...

Context — Cancellation & Deadlines

Every production Go API passes a context.Context for timeout/cancellation.

▾
🐹 context.Context
import ( "context" "time" ) // Always pass context as FIRST parameter func fetchUser(ctx context.Context, id int) (*User, error) { // Pass ctx to all downstream calls return db.QueryContext(ctx, "SELECT...", id) } // Timeout — cancel after 5 seconds ctx, cancel := context.WithTimeout( context.Background(), 5*time.Second, ) defer cancel() // always defer cancel! user, err := fetchUser(ctx, 123) // HTTP handlers get context from request func handler(w http.ResponseWriter, r *http.Request) { ctx := r.Context() fetchUser(ctx, 123) }

Clean Architecture & Design Patterns

Repository pattern, dependency injection, hexagonal architecture in Go.

▾
🐹 Repository Pattern
// Define interface (domain layer)
type UserRepository interface {
  FindByID(ctx context.Context, id int) (*User, error)
  Create(ctx context.Context, u *User) error
}

// Postgres implementation
type postgresUserRepo struct{ db *sql.DB }

func (r *postgresUserRepo) FindByID(ctx context.Context, id int) (*User, error) {
  // SQL query...
}

// Service depends on INTERFACE not implementation
type UserService struct {
  repo UserRepository  // can swap with mock!
}

// Constructor injection
func NewUserService(repo UserRepository) *UserService {
  return &UserService{repo: repo}
}
💡
Project structure used in production:
/cmd — entry points  |  /internal — private code  |  /pkg — public libraries  |  /api — proto/swagger definitions
